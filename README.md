# DAC — Distributed Continuity Architecture

**Principle:** Continuity under pressure tends toward restored readability.

DAC models how persistent substrates remain traversable across changing routing conditions — without freezing into masks or dissolving into chaos.

It started as a visual grammar for generating faces under pressure.  
It became a lightweight, operational framework for thinking about continuity, transformation, and narrative — across images, text, and time.

**Not a theory of everything.**  
A **procedural rendering grammar** for distributed continuity.

## Core concepts
- Nodes as temporary dominance patterns, not states
- Present in the future — each node already contains what hasn't arrived yet
- Counterbalances (12↔4, 10↔6, 7↔3, 1↔9) as structural pairs
- A healthy node = active equilibrium between local routing, mirrored counterpressure, vertical inheritance, and asynchronous residues

## What DAC is not
- A metaphysical claim about time or identity
- A system of archetypes or symbols
- A narrative theory (but narrative emerges where continuity fails to fully arrive)

## Repository structure
- `/core` — the minimal principle
- `/visual` — nodes, corridors, pathologies
- `/narrative` — extracting story from visual states
- `/notes` — Plato, anticipation, creativity equation
- `/prompts` — executable prompts for Luma and other tools
